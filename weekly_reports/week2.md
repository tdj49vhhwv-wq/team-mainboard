## 2. 企业清单获取方法

- **使用网站**：
  - 主源：上海证券交易所科创板官网 https://kcb.sse.com.cn（IPO项目 → 已上市 → 筛选2025年）
  - 补充：巨潮资讯网 https://www.cninfo.com.cn（证监会指定披露平台）
  - 校验：同花顺iFinD金融终端 + 东方财富Choice终端

- **检索路径**：
  ```
  上交所科创板首页 → IPO项目 → 已上市公司 → 筛选"上市年份:2025"
  → 获取列表: 公司全称、股票代码、上市日期、发行价格
  → 巨潮资讯逐家验证招股说明书存在性
  → iFinD/Choice导出数据交叉比对
  ```

- **是否代码抓取**：否。企业清单通过金融终端导出 + 官方网页人工确认，非爬虫抓取。招股书URL获取使用代码辅助搜索（`code/02_fetch_prospectus_urls/fetch_urls.py`）

- **是否人工核对**：是。每家公司在上交所官网手动确认上市日期，在巨潮资讯手动验证至少1份招股说明书存在

- **交叉核对来源**：四个来源交叉核对，5个字段（公司全称、股票代码、上市日期、发行价格、招股书存在性），19家全部一致

## 5. 本市场披露特点

- **常见章节名称**：
  - 主章节："第五节 发行人基本情况"（与主板/创业板格式统一）
  - 子章节："二、发行人设立情况" → "（三）股本演变/历次股本变化"
  - 科创板特有："符合科创板定位"、"科创属性"、"尚未盈利/累计未弥补亏损"
  - 红筹公司："红筹架构搭建"、"红筹架构拆除"、"VIE协议控制"

- **常见关键词**：
  ```
  股本演变 | 历次增资 | 股权转让 | 注册资本增加
  整体变更 | 吸收合并 | 红筹架构 | VIE
  特别表决权 | AB股 | 员工持股平台 | 科创属性
  ```

- **常见表格**：
  - 股本演变概图（mermaid flowchart）：展示公司设立以来的增资/转让/改制路径
  - 历次增资情况表：增资时间、增资方、增资金额、增资后注册资本
  - 股东持股情况表：股东名称、持股数、持股比例、关联关系
  - 对赌协议情况表（科创板特有）：投资方、协议内容、触发条件、解除情况

- **与公共样本相比的变化**：
  | 差异点 | Week 1 公共样本 | Week 2 科创板专攻 |
  |------|------|------|
  | 样本范围 | 8家（四大板块混合） | 19家（纯科创板） |
  | 融资轮次 | 平均3-4轮 | 半导体公司可达6+轮 |
  | 特殊架构 | 仅影石1家有VIE | 约5家有红筹/VIE历史 |
  | 上市标准 | 标准一为主 | 标准一/四/五/六混合（未盈利企业需要额外处理） |
  | 投资方类型 | VC/PE为主 | 新增：政府大基金（国家集成电路基金）、产业资本（中芯国际等） |

## 7. 本周形成的市场规则

- **企业List规则** (`source_notes/2025_company_list_source.md`)：
  - 四源交叉核对流程（上交所科创板 + 巨潮资讯 + iFinD + Choice）
  - 纳入标准：2025年科创板挂牌、688xxx、正式交易
  - 排除标准：注册未挂牌、终止审核、转板公司
  - sample_id编号规则：STAR001-STAR019按上市日期排序

- **招股书版本规则** (`source_notes/version_rules.md`)：
  - 优先级：申报稿 > 上会稿 > 注册稿 > 正式稿
  - 理由：申报稿融资历史最完整，正式稿可能因篇幅限制删减细节
  - 同一公司多版本取最新申报稿

- **章节定位规则** (`source_notes/star_market_locate_rules.md`)：
  - 主定位：第五节 → 二、发行人设立情况 → （三）股本演变
  - 降级搜索：关键词（增资/股权转让/出资方式）全文匹配
  - 科创板三种特殊场景处理：
    1. 红筹架构：融资分散在"搭建"和"拆除"两个段落，需合并
    2. 标准五医药：扩展搜索可转债/优先股/授权引进
    3. 半导体多轮次：逐轮次识别A/B/C/D轮，避免合并

- **抽取字段规则** (沿用Week 1 Schema)：
  - Event 18字段：event_order → confidence
  - Investor 8字段：investor_original_name → exit_status_before_ipo
  - date_type枚举：工商变更日 / 股东会决议日 / 协议签署日 / 未说明
  - event_type枚举：增资 / 股权转让 / 增资及股权转让 / 其他
  - investor_type枚举：VC / PE / 产业资本 / 自然人 / 员工持股平台 / 政府基金 / 其他 / 无法判断
  - confidence枚举：high（原文明确）/ medium（有证据不完整）/ low（推测为主）

- **文件排除规则**（详见 `source_notes/2025_company_list_source.md` §6）：
  - 12类非目标文件关键词排除清单
  - 排除判断流程图（文件名 → 排除关键词 → PDF页数 → 通过）
  - 人工复核触发条件（页数异常 / 搜索结果唯一 / 公司名变更）

- **PyMuPDF降级解析规则** (Week 2新增)：
  - MinerU不可用时使用PyMuPDF (`fitz`) 作为降级方案
  - PyMuPDF输出为 `## 第N页` 格式，不保留文档章节层级
  - 章节定位需要从文本中搜索 `第X节 标题` 模式（而非依赖markdown heading）
  - 全文关键词搜索降级策略：`发行人基本情况` → `股本演变` → `历次增资`
  - 已知局限：红筹/VIE架构公司（影石创新、昂瑞微等）融资历史分散在多个段落，PyMuPDF无法有效合并
  - 建议：优先使用MinerU解析；PyMuPDF仅作为过渡方案，解析后需人工验证章节完整性

- **event_type判定优先级规则** (Week 2新增)：
  - 优先级顺序：增资 > 股权转让 > 吸收合并 > 整体变更 > 设立 > 改制
  - 原因：文本中"未分配利润转增注册资本"同时匹配"增资"和"改制"关键词，按优先级应判为增资
  - 避免关键词误判：`转增` ≠ `改制`，`股权激励` ≠ `股权转让`
