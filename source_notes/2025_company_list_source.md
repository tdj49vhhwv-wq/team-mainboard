### 5.2 自动获取方法（批量）

使用项目脚本 `code/02_fetch_prospectus_urls/fetch_urls.py`：

```bash
# 方式一：手动填充已知URL（适用于已有确定链接的公司）
python3 code/02_fetch_prospectus_urls/fetch_urls.py --manual

# 方式二：通过巨潮资讯API自动搜索
python3 code/02_fetch_prospectus_urls/fetch_urls.py --search

# 方式三：两者结合
python3 code/02_fetch_prospectus_urls/fetch_urls.py --manual --search
```

**脚本工作原理**：
1. 读取 `company_lists/week2_2025_company_list.csv`
2. 对 `prospectus_url` 为空的行，以股票代码+公司全称为关键词
3. 调用巨潮资讯搜索API (`hisAnnouncement/query`)，筛选类别 `category_ndbg_szsh`（招股说明书）
4. 在返回结果中按文件名规则过滤（见第6节排除规则）
5. 按版本优先级选择最佳结果：申报稿 > 上会稿 > 注册稿 > 正式稿
6. 提取PDF链接（`static.cninfo.com.cn` 域名）和发布日期
7. 写回CSV的 `prospectus_title`, `prospectus_url`, `prospectus_version`, `prospectus_date` 字段

## 6. 如何排除非目标文件

### 6.1 目标文件定义

**目标文件**：公司首次公开发行股票并在科创板上市的**完整招股说明书**（或招股意向书）。

### 6.2 排除文件类型及关键词

在巨潮资讯搜索结果中，同一股票代码下可能出现多种公告文件，必须按以下规则逐一排除：

| 序号 | 排除文件类型 | 文件名关键词 | 排除原因 | 典型示例 |
|:---:|------|------|------|------|
| 1 | **上市公告书** | `上市公告书` | 仅在上市日发布，不含完整融资历史，篇幅短（<50页） | "XX公司首次公开发行股票科创板上市公告书" |
| 2 | **发行公告** | `发行公告` | 仅含发行安排（申购日期、发行价），无公司历史信息 | "XX公司首次公开发行股票并在科创板上市发行公告" |
| 3 | **问询函回复/问询回复** | `问询函`、`问询回复`、`反馈意见回复`、`落实函` | 交易所审核问询的回复文件，不含融资历史 | "XX公司及保荐机构关于审核问询函的回复" |
| 4 | **提示性公告** | `提示性公告`、`提示性` | 上市前风险提示，无实质内容 | "XX公司首次公开发行股票并在科创板上市提示性公告" |
| 5 | **招股说明书摘要** | `摘要` | 为简化版，篇幅大幅缩减（通常<50页），多数不保留完整历史沿革 | "XX公司首次公开发行股票并在科创板上市招股说明书摘要" |
| 6 | **招股意向书摘要** | `意向书摘要` | 同上，摘要版 | "XX公司首次公开发行股票并在科创板上市招股意向书摘要" |
| 7 | **补充法律意见书** | `法律意见书`、`补充法律意见` | 律所出具的法律意见，非公司招股书 | "XX律所关于XX公司IPO的补充法律意见书" |
| 8 | **审计报告** | `审计报告`、`财务报告` | 会计师出具，不含融资历史 | "XX公司2022-2024年度审计报告" |
| 9 | **发行保荐书** | `保荐书`、`上市保荐书` | 保荐机构出具，不含融资历史 | "XX证券关于XX公司科创板上市的发行保荐书" |
| 10 | **公司章程/制度文件** | `公司章程`、`议事规则` | 制度文件，不含融资历史 | "XX公司章程（草案）" |
| 11 | **发行结果公告** | `发行结果`、`中签率`、`中签结果` | 发行流程公告 | "XX公司首次公开发行股票并在科创板上市发行结果公告" |
| 12 | **投资风险特别公告** | `投资风险`、`特别公告` | 风险提示 | "XX公司首次公开发行股票并在科创板上市投资风险特别公告" |

### 6.3 排除判断流程

```
搜索结果 → 
  ├─ 文件名包含"招股说明书"或"招股意向书"?
  │   ├─ 否 → 排除，检查下一条
  │   └─ 是 → 文件名包含排除关键词?
  │       ├─ 是（摘要/问询/回复/法律意见/审计/保荐/公告）→ 排除
  │       └─ 否 → 检查PDF页数
  │           ├─ < 300页 → 可能是摘要或简化版，标记人工复核
  │           ├─ 300-800页 → ✓ 符合要求
  │           └─ > 800页 → 可能含大量附件，人工判断
  └─ 排除
```

### 6.4 代码实现（fetch_urls.py中的过滤逻辑）

```python
# 必须包含
if not re.search(r"招股(?:意向)?书", title):
    continue  # 排除

# 必须排除的关键词
EXCLUDE_PATTERNS = [
    r"摘要",           # 招股说明书摘要
    r"问询",           # 问询函/问询回复
    r"回复",           # 反馈意见回复
    r"上市公告",       # 上市公告书
    r"发行公告",       # 发行公告
    r"提示性",         # 提示性公告
    r"法律意见",       # 补充法律意见书
    r"审计报告",       # 审计报告
    r"保荐书",         # 发行保荐书/上市保荐书
    r"章程",           # 公司章程
    r"发行结果",       # 发行结果公告
    r"中签",           # 中签率/中签结果
    r"投资风险",       # 投资风险特别公告
]

for pattern in EXCLUDE_PATTERNS:
    if re.search(pattern, title):
        continue  # 排除
```

### 6.5 版本选择（在通过排除规则后）

通过排除规则筛选后，可能仍有多个版本的招股说明书（申报稿、上会稿、注册稿、正式稿）。选择规则详见 `source_notes/version_rules.md`：

| 优先级 | 版本 | 特征 | 融资历史完整度 |
|:---:|------|------|:---:|
| 1 | 申报稿 | 文件名含"申报稿" | 最完整 |
| 2 | 上会稿 | 文件名含"上会稿" | 较完整 |
| 3 | 注册稿 | 文件名含"注册稿" | 可能简化 |
| 4 | 正式稿 | 无特殊标注 | 可能删减细节 |

### 6.6 人工复核触发条件

以下情况需人工打开PDF确认：
- 文件名同时包含"招股说明书"和疑似排除关键词
- PDF页数不在 300-800 范围
- 搜索结果中仅有1条符合条件的招股说明书（无法交叉验证版本）
- 公司名称变更导致搜索匹配异常
