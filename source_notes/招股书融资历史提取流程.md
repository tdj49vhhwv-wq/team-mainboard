# 招股书融资历史提取流程

## 1. 企业清单如何建立？

- **数据源**：从交易所官网（北京证券交易所、巨潮资讯）获取 IPO 排队/已上市企业列表
- **来源渠道**：
    - 北京证券交易所：https://www.bse.cn/index.html?eqid=ab9dc0010000f3ac00000002650013c5
    - 巨潮资讯网：https://www/cninfo.com.cn/new/index
  - 方式：爬取或从金融终端导出企业名称、代码、受理日期等字段，形成 CSV/Excel 清单
  - 字段建议：企业名称、股票代码、受理日期、审核状态、招股书链接

  3. 如何判断文件是不是目标招股说明书？
  
  - 文件名关键词匹配：包含"招股说明书"、"招股意向书"、"招股书"
  - 文件内容特征：
    - 首页通常有"XX证券股份有限公司"、"招股说明书"等字样
    - 包含典型章节如"释义"、"发行人基本情况"、"业务与技术"、"财务会计信息"
    - 排除：问询回复函、上市公告书、发行公告等
  - PDF页数：招股书通常在 300-800 页，太短（<100页）或太长（>1000页）可能是其他文件
  - 实际做法：先用文件名初筛，再用 PDF 前几页内容正则或 AI 确认

  5. 融资历史通常出现在哪些章节？
  
  在中文招股书中，融资历史主要出现在：

  1. "发行人基本情况" — 最核心来源，通常会有一段"发行人设立以来的股本演变情况"或"历史沿革"，详细列出历次增资/股权转让
  2. "发行人设立情况及报告期内的股本和股东变化情况" — 专门小节
  3. "历次增资及股权转让情况" — 直接对应融资历史表格
  4. "股东情况" — 有时会附带融资信息
  5. "财务会计信息" — 部分融资细节（估值、金额）散落在此

  关键词提示：增资、股权转让、融资、股本演变、历史沿革、估值、投后、每注册资本

  6. 如何截取候选文本？
  
  分两步：

  粗定位：按章节标题匹配，截取"发行人基本情况"整个大节的文本（从章节标题到下一个同级别标题之间）

  精确定位：在粗截取的文本中，用关键词定位到具体段落：
  - 增资|股权转让|引入.*投资|融资 匹配到起始位置
  - 向前后各扩展 N 段/字（如前后各 2000 字或到子章节边界）
  
  import re
  # 找到章节区间
  chapter_match = re.search(r'(发行人基本情况|历史沿革|股本.*演变)', text)
  # 截取候选段落
  candidate = text[start:end]

  也可以把整个章节喂给AI，让AI自己找融资历史相关段落。

  7. 如何让AI输出JSON？
  
  核心在 Prompt 设计：

  请根据以下融资历史文本，提取融资事件，严格按照JSON格式输出：

  [
    {
      "round": "A轮",
      "date": "2020-06",
      "amount": "5000万元",
      "valuation": "投后5亿元",
      "investors": ["红杉资本", "高瓴资本"]
    }
  ]

  要求：
  - 只输出JSON数组，不要任何其他文字
  - 日期格式统一为YYYY-MM
  - 金额单位统一为"万元"

  更保险的做法：
  - 使用 JSON Mode / Structured Output（OpenAI/Anthropic 都支持）
  - 使用 response_format={"type": "json_object"} 强制 JSON 输出
  - 在 prompt 末尾强调 "只输出JSON，不要markdown代码块包裹"

  8. 如何校验JSON是否合格？
  
  import json

  def validate_json(json_str):
      # 1. 能解析
      try:
          data = json.loads(json_str)
      except json.JSONDecodeError as e:
          return False, f"JSON解析失败: {e}"

      # 2. 检查结构（必须是数组）
      if not isinstance(data, list):
          return False, "JSON必须是数组"

      # 3. 检查必填字段
      required = ["round", "date", "amount", "investors"]
      for i, item in enumerate(data):
          for field in required:
              if field not in item:
                  return False, f"第{i}条记录缺少字段: {field}"

      # 4. 检查字段类型
      for item in data:
          if not isinstance(item.get("investors"), list):
              return False, "investors必须是数组"

      return True, "OK"

  校验维度：
  - 语法校验：json.loads 能否解析
  - Schema校验：必填字段、字段类型、枚举值是否合法
  - 业务校验：日期格式、金额合理性、不重复等

  9. 失败样本如何记录？
  
  建议用 CSV 或 JSONL 记录，保留完整上下文方便复查：

  import csv
  from datetime import datetime

  def log_failure(file_path, error_type, error_detail, raw_output=None):
      record = {
          "timestamp": datetime.now().isoformat(),
          "file": file_path,
          "error_type": error_type,      # parse_failed / json_invalid / schema_mismatch / empty_result
          "error_detail": error_detail,
          "raw_output": raw_output or "", # AI原始输出，方便回溯分析
      }
      # 写入 CSV 或 JSONL
      with open("failed_samples.jsonl", "a") as f:
          f.write(json.dumps(record, ensure_ascii=False) + "\n")

  失败分类建议：
  - pdf_parse_error — PDF解析失败
  - chapter_not_found — 没找到目标章节
  - json_parse_error — AI输出不是合法JSON
  - schema_mismatch — JSON字段缺失/类型错误
  - empty_result — AI认为没有融资历史但人工标注有
  - wrong_extraction — 提取得了但数据错误（需要人工review发现）
  
  记录好失败样本后，可以定期做 bad case分析，优化 prompt 或解析策略。
